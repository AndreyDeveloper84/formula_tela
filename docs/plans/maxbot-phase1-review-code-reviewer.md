## BLOCKING

### #1 — `webhook_secret` читается из env, но не передаётся в `dp.handle_webhook()`
`maxbot/main.py:57-62`

`MaxBotConfig.webhook_secret` хранит значение, в логе `bool(cfg.webhook_secret)`, но при вызове `dp.handle_webhook(...)` параметр `secret_token` (или эквивалент в maxapi 1.0.0) не передаётся. Любой может слать боту фиктивные апдейты через публичный URL `/api/maxbot/webhook/`.

**Fix:** проверить сигнатуру `handle_webhook` в SDK 1.0.0 и добавить аргумент. Если SDK не поддерживает — убрать поле из конфига и `.env.example` чтобы не создавать ложное ощущение безопасности.

### #2 — Async-тесты без `transaction=True` → broken test isolation
`tests/maxbot/test_personalization.py:6`

`pytestmark = pytest.mark.django_db` без `transaction=True`. С `asyncio_mode=auto` pytest-asyncio выполняет async-тесты в отдельном event loop. `sync_to_async(thread_sensitive=True)` идёт в главном потоке, но `django_db` без `transaction=True` использует savepoints/TestCase — некорректно в async-контексте. Тесты могут быть зелёными локально и красными в CI.

**Fix:**
```python
pytestmark = pytest.mark.django_db(transaction=True)
```

## MAJOR

### #3 — `HelpArticle` без кастомного QuerySet — нарушение проектного паттерна
`services_app/models.py:1051-1067`

Весь проект использует `XxxQuerySet.active()` (ServiceQuerySet, ReviewQuerySet, MasterQuerySet в `managers.py`). HelpArticle имеет `is_active`, но нет `HelpArticleQuerySet.active()` — handlers Фазы 2 неизбежно напишут `.filter(is_active=True)` вместо `.active()`.

**Fix:** добавить в `services_app/managers.py`:
```python
class HelpArticleQuerySet(models.QuerySet):
    def active(self):
        return self.filter(is_active=True)
```
Подключить: `objects = HelpArticleQuerySet.as_manager()`.

### #4 — `update_context`/`append_to_context` race condition
`maxbot/personalization.py:51-69` *(дублирует ln-624 H1)*

GET → modify in Python → SAVE без транзакции/lock. Тихая потеря данных при concurrent webhook'ах.

**Fix:**
```python
with transaction.atomic():
    user = BotUser.objects.select_for_update().get(pk=bot_user_id)
    user.context.update(updates)
    user.save(update_fields=["context", "last_seen"])
```
Не критично для Фазы 1 (трафик мал), но обязательно до Фазы 2 (FSM с несколькими шагами).

### #5 — Миграция 0057 сгенерирована **Django 6.0.3** в проекте на **5.2**
`migrations/0057_botuser_helparticle_bookingrequest_source_and_more.py:1`
